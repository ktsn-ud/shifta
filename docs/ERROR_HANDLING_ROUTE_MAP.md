# エラーハンドリング ルート別メモ

最終更新日: 2026-04-08  
対象: Next.js App Router の全ルート（`nextjs_index -> get_routes` で確認）

## 1. 運用ルール（メモ更新方針）

- ルート追加・API追加時はこのファイルに同日追記する。
- `fallback` 文言だけでなく「ユーザーが次に何をすべきか」を必ず明記する。
- セキュリティ観点で、UIには生の例外詳細（内部エラー文字列・スタック）を表示しない。
- 画面側は以下の優先順で対処法を表示する。
  1. 入力修正（バリデーション）
  2. 再実行（通信・一時障害）
  3. 再ログイン/権限確認
  4. Google Calendar 再設定（必要時）

## 4. 表示方針（実装済み）

- 画面内エラーと `sonner` の説明文は、共通変換で「対処法付きメッセージ」に統一。
- 生の `payload.error` や `error.message` の直接表示は避け、HTTPステータス/エラーコードに基づいて表示。
- Google同期関連は `requiresCalendarSetup` / `code` を使い、必要時はカレンダー再設定へ誘導。
- 予期しないレンダリング例外は `app/error.tsx` と `app/my/error.tsx` で対処方法を表示。
