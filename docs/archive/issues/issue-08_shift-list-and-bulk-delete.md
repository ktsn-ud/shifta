# [issues-8] シフト一覧ページを作成し、一括削除を可能とする

## 概要

シフト一覧ページ（/my/shifts/list）を作成し、メニューの「ダッシュボード」と「シフト確定」の間にナビゲーションを表示する。

シフト一覧はテーブルの形で表示をし、以下のカラムを作成する。

左から

- 選択（チェックボックス）
- 日付
- 時間
- 勤務先（丸バッジと勤務先名：塾タイプの事務シフトは（事務）をつける）
- 休憩時間
- 給与

デフォルトでは日付/時間の昇順で表示をする。それぞれのカラムで並び替え可能に。デフォルト表示に戻すボタンも必要。

また、ダッシュボードと同様に、月ごとに切り替えて表示できるようにする。今月に戻るボタンも必要。

各レコードをクリックすれば該当シフトの編集画面に飛べるようにする。パスクエリにmonthを含めることで、もとの月表示に戻れるようにする。（ダッシュボードと同様）

選択ボタンによって一括選択されたシフトを一括削除できるような機能を実装する。

## 現状分析（2026-03-31）

- `/my/shifts/list` ルートは未実装（`nextjs_index` の `get_routes` に存在しない）。
- 左メニューは `components/app-sidebar.tsx` で定義されており、現状は `ダッシュボード -> シフト確定 -> 給与サマリー...` の順。シフト一覧の導線がない。
- パンくずは `components/site-header.tsx` の `buildBreadcrumbs` で `section === "shifts"` を分岐しているが、`list` は未対応。
- シフト月次データ取得は `hooks/use-month-shifts.ts` + `GET /api/shifts?startDate&endDate&includeEstimate=true` で実装済み（`breakMinutes` / `estimatedPay` / `workplace` を取得できる）。
- シフト削除APIは単体削除のみ（`DELETE /api/shifts/[id]`）。複数IDの一括削除APIは未実装。
- 勤務先表示の「塾タイプ事務 = （事務）」付与ロジックは `components/calendar/ShiftListModal.tsx` 内のローカル関数のみで、再利用できる共通化はされていない。
- シフト編集後の戻り先は `ShiftForm` の `returnMonth` で `/my?month=YYYY-MM` 固定。シフト一覧ページ起点で編集した場合に一覧へ戻す仕組みは未実装。

### 4. 一括削除APIの追加

- 追加候補: `DELETE /api/shifts`（`app/api/shifts/route.ts` に `DELETE` ハンドラ追加）

  - 検証: 空配列不可、重複除去、最大件数制限（例: 100）
  - 権限: 全IDがログインユーザー所有であることを検証

  - 連携: 既存単体削除と同様に Google Calendar 削除を `after()` で非同期実行

- 既存の `verifyMutationRequest` による CSRF 検証を適用する。
