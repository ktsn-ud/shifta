# UXリデザイン実行計画（デザイン全面刷新）

## 1. 目的

本計画は、既存機能の**内容・業務ロジックを変えず**、UXと情報設計のみを全面的に再設計するための実行手順である。

達成目標:

- 情報の優先順位が即座に読み取れるレイアウトへ統一する
- 高頻度操作（シフト登録・一覧確認・給与確認）を最短動線化する
- 既存の shadcn/ui コンポーネントを再利用しつつ、見た目と体験を再設計する

## 2. 設計方針

- データ密度と可読性の両立: 「見出し > 要約値 > 操作 > 詳細」の順で視線が流れる構造に統一する
- 操作の予測可能性: 画面間で同じ位置・同じ見た目・同じ挙動の操作を保証する
- 状態の明示: loading / empty / error / success を一貫した見た目で提示する
- 低認知負荷: 1画面内の同時判断数を削減し、グルーピングと余白で意味を分離する
- アクセシビリティ: コントラスト、フォーカス可視性、タップ領域、キーボード操作を維持/向上する

## 3. 対象範囲

対象（デザイン面）:

- グローバルトークン（配色、余白、半径、影、フォーカス）
- 共通UIプリミティブ（Button/Card/Input/Select/Table/Dialog/Badge/Tabs など）
- アプリシェル（Sidebar/Header/Breadcrumb/ページ余白）
- 主要画面（ログイン、ダッシュボード、シフト一覧、勤務先管理、給与サマリー、給与詳細）

非対象（今回変更しない）:

- API契約、DBスキーマ、給与計算ロジック、外部連携仕様
- 業務上の意味変更を伴う文言改訂

## 4. 実装タスク

### T1: デザイン基盤刷新

目的:

- 新しい情報デザインの土台を全画面で共有する

作業:

- `app/globals.css` のトークン体系を再定義（背景階層・境界・アクセント・状態色）
- shadcn/uiプリミティブの見た目を統一（密度、角丸、影、フォーカス）
- 状態表示コンポーネント（empty/error/loadingの見た目）を共通化

成果:

- どの画面でも同じ視覚言語が機能する状態

### T2: ナビゲーションとページ骨格刷新

目的:

- 迷わない導線と安定した情報フレーミングを作る

作業:

- `components/app-sidebar.tsx` と `components/site-header.tsx` を再設計
- アクティブ状態・階層表示・主要操作の視認性改善
- ページ共通ヘッダー/説明/アクション配置パターンを統一

成果:

- 画面遷移先で文脈が途切れず、現在地と次操作が明確な状態

### T3: 主要業務画面のUX再構築

目的:

- 高頻度タスクを短時間で処理できる画面に再設計

作業:

- ダッシュボード: 要約カードと次アクションの優先順を整理
- シフト一覧: 検索・選択・一括操作の視線動線最適化
- 勤務先管理: 一覧情報の意味単位を再編し、操作ボタン群を整理
- 給与サマリー/給与詳細: 集計理解に必要な情報順へ並べ替え
- ログイン: 信頼感と開始導線の明確化

成果:

- 主要業務を迷いなく処理できる画面群

## 5. 体制（サブエージェント分担）

- Agent A: T1（デザイン基盤）
- Agent B: T2（ナビゲーション/シェル）
- Agent C: T3（主要画面）
- メインエージェント（私）: 差分監査、統合修正、品質確認、コミット

## 6. 品質ゲート（各タスクごと）

コード変更を含む各タスクの完了条件:

1. `pnpm exec tsc --noEmit`
2. `pnpm lint`
3. `pnpm test`
4. `pnpm format`

上記通過後に差分監査を実施し、1目的1コミットで記録する。

## 7. コミット計画

- Commit 1: `docs`（本計画書の追加）
- Commit 2: T1 デザイン基盤
- Commit 3: T2 ナビゲーション/シェル
- Commit 4: T3 主要画面UX
- 必要に応じて監査修正コミットを追加

## 8. リスクと対策

- リスク: 画面ごとの独自スタイルが残り統一感が崩れる
  - 対策: まず共通コンポーネントを刷新し、個別画面差分を最小化
- リスク: 見た目変更による操作回帰
  - 対策: 既存テスト通過 + 主要導線の手動確認（Next.js runtime error 監視）
- リスク: 変更範囲が広くレビュー困難
  - 対策: タスク単位コミットと監査ログで追跡可能性を確保
