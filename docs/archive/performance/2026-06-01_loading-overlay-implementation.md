# ローディング表示のオーバーレイ化 実装ドキュメント（2026-06-01）

## 1. 実装目的

月遷移や再取得時に既存コンテンツを消さず、表示を維持したままローディングオーバーレイを重ねることで、
画面の途切れとレイアウトジャンプを防止する。

## 2. 実装範囲

### 対象

- 置換型ローディングUIを再取得時のみオーバーレイ型へ移行する
- 初回表示と再取得のローディング状態を分離する
- 優先度順に対象画面へ適用する

### 非対象

- 変更検知（revision watch）
- optimistic update / prefetch
- バックグラウンド同期の設計変更
- API・DBスキーマ変更

## 3. 実装ルール

- 初回表示（データ未取得）は skeleton / full spinner を使用する
- 2回目以降の再取得はコンテンツを unmount せずオーバーレイ表示にする
- オーバーレイ中は対象領域に `aria-busy=true` を付与する
- 必要に応じて `pointer-events` を制御し二重操作を防ぐ
- レイアウト高さを維持し、ローディング中のジャンプを防ぐ
- オーバーレイ内に `最新データを更新中...` を表示する

## 4. 適用優先度

### 優先1

- `/my` ダッシュボード（`MonthCalendar`）
- `/my/shifts/list` 一覧テーブル
- `/my/shifts/bulk` 日付カレンダー

### 優先2

- `/my/summary`
- `/my/payroll-details/monthly`
- `/my/payroll-details/workplace-yearly`
- `/my/shifts/confirm`

## 5. 実装方針

### 状態分離

- `isInitialLoading`: 初回ロード専用
- `isRefreshing`: 再取得専用（オーバーレイ表示判定）

`useMonthShifts` などのデータ取得フックで、`isLoading` と再取得状態（`isFetching` 相当）を分離して公開する。

### UI部品

- 共通 `LoadingOverlay` コンポーネントを新設する
- 既存コンテンツを包むコンテナにオーバーレイを重ねる
- オーバーレイ表示中もコンテンツ本体は描画維持する

## 6. 完了条件

- 月遷移/再取得時に主要コンテンツが消えない
- オーバーレイで更新中であることが視認できる
- 初回表示時は従来どおり全体ローディングが機能する
- 画面ジャンプや二重操作の問題が発生しない
