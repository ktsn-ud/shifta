# シフト管理アプリ

## 統合設計書

---

# 1. システム概要

本システムは  
**個人用シフト・給与概算管理Webアプリ**である。

主な機能

- シフト登録・編集・削除
- 月間カレンダー表示
- 概算給与計算・集計
- Google Calendar同期

---

# 2. システム構成

**アーキテクチャレイヤー**

| レイヤー           | 技術スタック           | 説明                                     |
| ------------------ | ---------------------- | ---------------------------------------- |
| **フロントエンド** | Browser + React 19     | ユーザーブラウザで実行される UI レイヤー |
| **バックエンド**   | Next.js App Router     | API ルートと Server Components           |
| **ORM層**          | Prisma + Neon Adapter  | データベース抽象化レイヤー               |
| **データベース**   | Neon（PostgreSQL互換） | サーバーレス PostgreSQL データベース     |
| **外部連携**       | Google Calendar API    | カレンダー同期機能                       |

**認証**

- NextAuth.js を使用したメール認証
- 単一ユーザー対応

**通信フロー**

Browser (React UI) → Next.js Routes → Prisma ORM → Neon DB

外部連携: Neon ← → Google Calendar API

| `/my`                | SCR_002 | ダッシュボード | ホーム画面                                      | 必須     |
| `/my/calendar`       | SCR_003 | カレンダー     | 月間シフト表示                                  | 必須     |
| `/my/shifts/confirm` | SCR_015 | シフト確定     | シフト確定ページ                                | 必須     |
| `/my/summary`        | SCR_007 | 給与集計       | 給与集計表示                                    | 必須     |
| `/my/workplaces`     | SCR_008 | 勤務先管理     | 勤務先CRUD（一覧）                              | 必須     |
| `/my/workplace`      | SCR_008 | 勤務先管理     | 旧URL（`/my/workplaces` へリダイレクト）        | 必須     |
| `/my/payroll`        | SCR_010 | 給与ルール管理 | 勤務先選択（給与ルール管理画面への入口）        | 必須     |

- 勤務先:
  - `/my/workplaces/new`（SCR_009 作成）
  - `/my/workplaces/:workplaceId/edit`（SCR_009 編集）
- 給与ルール:
  - `/my/workplaces/:workplaceId/payroll-rules`（SCR_010 一覧）
  - `/my/workplaces/:workplaceId/payroll-rules/new`（SCR_011 作成）
  - `/my/workplaces/:workplaceId/payroll-rules/:ruleId/edit`（SCR_011 編集）
- 時間割:
  - `/my/workplaces/:workplaceId/timetables`（SCR_012 一覧）
  - `/my/workplaces/:workplaceId/timetables/new`（SCR_013 作成）
  - `/my/workplaces/:workplaceId/timetables/:id/edit`（SCR_013 編集）

**ホームページ仕様**

- **`/my`**: ホームページ。デフォルトではカレンダーを表示する。
- **ログイン前**: 自動的に `/login` へリダイレクト
- **ログイン後**: `/my` へリダイレクト（初期表示はカレンダー）

# 3. ドメインモデル

本システムは単一ユーザー専用である。各ドメインモデルの関連性は以下の通り。

**エンティティ関連性**

- **User** (ユーザー)
  - **Workplace** (勤務先) - 複数登録可能
    - **PayrollRule** (給与ルール) - 複数、時系列管理
    - **Timetable** (時間割) - 複数、塾のみ
    - **Shift** (シフト) - 複数
      - **ShiftLessonRange** (授業コマ範囲) - 複数、LESSON タイプのシフトのみ

---

## 3.1 User

ログインユーザー

**説明**: NextAuth.js でログイン管理されるユーザー。本システムは単一ユーザー対応のため、通常1件のみ存在する。

| 属性      | 型       | 説明                         |
| --------- | -------- | ---------------------------- |
| id        | UUID     | ユーザーの一意識別子         |
| email     | string   | メールアドレス（ログインID） |
| createdAt | datetime | アカウント作成日時           |

**制約条件**

- email は一意（unique）
- email は有効なメール形式

**使用例**

- email: user@example.com
- createdAt: 2026-01-15T08:30:00Z

---

## 3.2 Workplace

勤務先（複数登録可能）

**説明**: ユーザーが勤務する企業・塾など。シフトの根拠となる組織。

| 属性      | 型       | 説明                                       |
| --------- | -------- | ------------------------------------------ |
| id        | UUID     | 勤務先の一意識別子                         |
| name      | string   | 勤務先名（例：「コンビニA」「塾B」）       |
| color     | string   | UIに表示する色（HEX形式）                  |
| type      | enum     | 勤務先の種別（GENERAL または CRAM_SCHOOL） |
| createdAt | datetime | 勤務先登録日時                             |

**制約条件**

- name は空でない
- color は HEX カラーコード（#RRGGBB）形式
- 1ユーザーあたり複数勤務先可能
- 削除時は関連する Shift, PayrollRule も削除される（CASCADE）

**使用例**

- name: "コンビニA"、type: GENERAL、color: "#FF5733"
- name: "英語塾"、type: CRAM_SCHOOL、color: "#3366FF"

**type の説明**

- **GENERAL**: 通常の勤務先（コンビニ、飲食店など）
- **CRAM_SCHOOL**: 塾（Timetable と ShiftLessonRange を使用）

---

## 3.3 PayrollRule

給与ルール履歴（勤務先ごとの時給管理）

**説明**: 勤務先ごとの時給・割増ルールを時系列管理。同一勤務先で複数ルールが存在し、シフト日に該当する有効なルールを自動選択する。

**ルール選択ロジック**: `startDate ≤ shift.date < endDate` を満たすルール中、`startDate` が最新（最大）のものを選択。同一勤務先内では適用期間の重複を許さない。

| 属性                   | 型      | 説明                                           |
| ---------------------- | ------- | ---------------------------------------------- |
| id                     | UUID    | ルールの一意識別子                             |
| workplaceId            | UUID    | 勤務先ID（外部キー）                           |
| startDate              | date    | ルール適用開始日                               |
| endDate                | date?   | ルール適用終了日（NULLの場合は現在有効）       |
| baseHourlyWage         | number  | 基本時給（円、小数2位まで。NORMAL/OTHER型用）  |
| perLessonWage          | number? | コマ給（円、小数2位まで。LESSON型用）          |
| holidayHourlyWage      | number? | 休日時給（NULLの場合は baseHourlyWage と同じ） |
| nightMultiplier        | number  | 深夜割増率（例：1.25）                         |
| dailyOvertimeThreshold | number  | 1日の所定時間（超過分が残業対象）              |
| overtimeMultiplier     | number  | 残業割増率（例：1.5）                          |
| nightStart             | time    | 深夜開始時刻（HH:MM、例：22:00）               |
| nightEnd               | time    | 深夜終了時刻（HH:MM、例：05:00）               |
| holidayType            | enum    | 休日判定タイプ                                 |

**制約条件**

- startDate と endDate が重複しない（同一勤務先内で）
- endDate は NULLか startDate より後
- baseHourlyWage > 0
- perLessonWage > 0
- nightMultiplier ≥ 1.0
- overtimeMultiplier ≥ 1.0
- dailyOvertimeThreshold > 0（時間、実数可）

**holidayType の説明**

- **NONE**: 休日判定なし（毎日同じ時給）
- **WEEKEND**: 土日を休日扱い
- **HOLIDAY**: 指定の祝日のみ休日扱い（将来拡張）
- **WEEKEND_HOLIDAY**: 土日と祝日を休日扱い

**使用例**

勤務先「コンビニA」:

- Rule 1: 2026-01-01 ～ 2026-03-31、baseHourlyWage=1100、nightMultiplier=1.25
- Rule 2: 2026-04-01 ～ NULL、baseHourlyWage=1150、nightMultiplier=1.25

勤務先「塾B」（CRAM_SCHOOL）:

- Rule 1: 2026-01-01 ～ 2026-03-31、perLessonWage=2000、nightMultiplier=1.0
- Rule 2: 2026-04-01 ～ NULL、perLessonWage=2500、nightMultiplier=1.0

## 3.4 Timetable

塾の時間割（CRAM_SCHOOL タイプの勤務先のみ使用）

**説明**: 塾の授業コマごとの開始・終了時刻を定義。通常期間と講習期間など複数のセットを管理可能。シフト登録（LESSON型）時に、選択したコマ範囲から`startTime/endTime`を自動計算する際に使用。

**用途**：

- LESSON型シフト時：「5限～7限」のように選択すると、該当コマの時刻を自動取得
- NORMAL型の場合も塾に登録可能（手入力、会議や事務作業など授業以外の業務を記録時に使用）

| 属性        | 型   | 説明                                                                    |
| ----------- | ---- | ----------------------------------------------------------------------- |
| id          | UUID | 時間割の一意識別子                                                      |
| workplaceId | UUID | 勤務先ID（CRAM_SCHOOL のみ、外部キー）                                  |
| type        | enum | 授業タイプ（NORMAL または INTENSIVE）                                   |
| period      | int  | コマ番号（1, 2, 3, ... など。typeごとに独立、例：通常期1限、講習期1限） |
| startTime   | time | コマの開始時刻（HH:MM）                                                 |
| endTime     | time | コマの終了時刻（HH:MM）                                                 |

**制約条件**

- workplaceId は type が CRAM_SCHOOL
- (workplaceId, type, period) は一意（type ごとに独立した period 番号体系を持つ）
- startTime < endTime
- endTime は翌日まで跨ぐ可能性あり（例：23:50 ～ 00:30）

**type の説明**

- **NORMAL**: 通常期間の授業
- **INTENSIVE**: 講習期間（集中講座・特別授業等）

同一勤務先・同一period内で、両typeのコマを登録可能（時期に応じて使い分け）。

**使用例**

塾「英語塾」の時間割（通常期と講習期で時間帯が異なる場合の例）:

通常期（type=NORMAL）:

- Period 1: 16:30 ～ 17:30
- Period 2: 17:40 ～ 18:40
- Period 3: 18:50 ～ 19:50

講習期（type=INTENSIVE）:

- Period 1: 15:00 ～ 16:20
- Period 2: 16:30 ～ 17:50
- Period 3: 18:00 ～ 19:20

---

## 3.5 Shift

シフト（実際の勤務）

**説明**: ユーザーが登録する実際の勤務シフト。勤務先ごと・日ごとに複数登録可能。Google Calendar と同期。

**LESSON型の特徴**：

- ユーザーは「5限～7限」のようにコマ範囲を選択
- Timetableから該当コマの時刻を自動計算 → `startTime/endTime` に格納
- ShiftLessonRange で「どのコマ範囲か」を記録
- Google Calendar には自動計算された時刻で登録される

**NORMAL型（塾での会議・事務作業など授業以外の業務）の特徴**

- ユーザーが時刻を手入力
- 時給で給与計算される（PayrollRule の baseHourlyWage を使用）

| 属性          | 型       | 説明                                    |
| ------------- | -------- | --------------------------------------- |
| id            | UUID     | シフトの一意識別子                      |
| workplaceId   | UUID     | 勤務先ID（外部キー）                    |
| date          | date     | シフトの日付                            |
| startTime     | time     | シフト開始時刻（HH:MM）                 |
| endTime       | time     | シフト終了時刻（HH:MM）                 |
| breakMinutes  | int      | 休憩時間（分単位、デフォルト0）         |
| shiftType     | enum     | シフトのタイプ（NORMAL, LESSON, OTHER） |
| isConfirmed   | boolean  | シフト確定フラグ（デフォルト false）    |
| googleEventId | string?  | Google Calendar のイベントID（同期用）  |
| createdAt     | datetime | シフト登録日時                          |

**制約条件**

- (workplaceId, date) ペアで同日内の時間帯重複は許可しない（バリデーション段階で警告）
- startTime < endTime（同日内）
- breakMinutes ≥ 0
- shiftType = LESSON の場合、ShiftLessonRange が1件以上存在
- shiftType = LESSON 以外の場合、ShiftLessonRange は存在しない

**shiftType の説明**

- **NORMAL**: 通常勤務（手入力。勤務先がCRAM_SCHOOLの場合は会議や事務作業など授業以外の業務も含む。時給で給与計算）
- **LESSON**: 塾の授業（CRAM_SCHOOL のみ。ShiftLessonRange で「何コマ目から何コマ目まで」を定義。時刻は Timetable から自動計算）
- **OTHER**: その他（自由入力）

**使用例**

- 2026-03-20、コンビニA、10:00 ～ 18:00、NORMAL、breakMinutes=60
- 2026-03-21、塾B、16:30 ～ 19:50、LESSON（Period 1～3）、breakMinutes=0

---

## 3.6 ShiftLessonRange

塾のコマ範囲（Shift.shiftType = LESSON の場合のみ）

**説明**: LESSON タイプのシフトが どのコマから どのコマまでをカバーするかを記録。Timetable のコマに対応する。複数コマの授業を1シフトで記録可能。

**注記**：`startPeriod ≤ endPeriod` の同じ `type`（NORMAL or INTENSIVE）のコマ範囲を指す。異なる `type` のコマを混在させることはできない。

| 属性        | 型   | 説明                                         |
| ----------- | ---- | -------------------------------------------- |
| id          | UUID | レコードの一意識別子                         |
| shiftId     | UUID | シフトID（外部キー）                         |
| startPeriod | int  | 開始コマ番号                                 |
| endPeriod   | int  | 終了コマ番号（startPeriodと同じ場合は1コマ） |

**制約条件**

- shiftId の Shift.shiftType = LESSON
- startPeriod ≤ endPeriod
- startPeriod, endPeriod は Timetable のコマの period に存在し、かつ同一 `type`（NORMAL or INTENSIVE）
- 1シフトあたり1件（複数コマの授業を1つのシフトで表現）

**使用例**

- Shift（2026-03-21、塾B、16:30 ～ 19:50、LESSON）
  → ShiftLessonRange: startPeriod=1, endPeriod=3

# 4. 主要ユースケース

---

## UC-001: シフト登録

**目的**: 新しいシフトをDB に登録し、Google Calendar に同期する。

**前提条件**

- ユーザーがログイン状態
- 勤務先が最低1件登録済み
- （LESSON の場合）塾の時間割が設定済み

**フロー**

1. ユーザーがカレンダーから日付をクリック
2. シフト入力フォーム表示（勤務先、シフトタイプなど）
3. ユーザーが情報を入力
   - **NORMAL型の場合**：開始時刻・終了時刻を手入力
   - **LESSON型の場合**：コマ範囲（e.g. Period 1～3）を選択 → サーバー側で Timetable から時刻を自動計算
4. クライアント側でバリデーション（必須項目、時刻の妥当性など）
5. バリデーション OK → POST /api/shifts（Shift + ShiftLessonRange を作成）
6. サーバー側で：
   - LESSON型の場合、Timetable から `startTime/endTime` を自動計算
   - Shift を INSERT → ShiftLessonRange を INSERT
7. DB 保存成功 → POST /api/calendar-sync にイベント作成を依頼（自動計算した時刻を使用）
8. Google Calendar API でイベント作成 → googleEventId を Shift に UPDATE
9. 成功レスポンス → 前画面へ戻る + 成功メッセージ表示

**エラーハンドリング**

- バリデーションエラー → UI でユーザーに指摘し、修正を促す
- Google Calendar API エラー → DB には保存済み、スタッシュして後で再同期（将来）

---

## UC-002: シフト編集

**目的**: 既存シフトを編集し、Google Calendar に反映する。

**前提条件**

- ユーザーがログイン状態
- 編集対象のシフトが存在

**フロー**

1. ユーザーがカレンダーまたはシフト一覧からシフトを選択
2. シフト詳細/編集フォーム表示（既存値をプリセット）
3. ユーザーが値を変更 → 「更新」ボタンをクリック
4. クライアント側でバリデーション
5. バリデーション OK → PUT /api/shifts/{id}（Shift + ShiftLessonRange を更新）
6. サーバー側で Shift を UPDATE + ShiftLessonRange を DELETE/INSERT
7. DB 更新成功 → PUT /api/calendar-sync/{googleEventId} でイベント更新
8. Google Calendar API でイベント更新
9. 成功レスポンス → 前画面へ戻る + 成功メッセージ表示

**エラーハンドリング**

- 同じシフト内での時間帯重複チェック（警告のみ）
- Google Calendar API エラー → DB は更新済み

---

## UC-003: シフト削除

**目的**: シフトを削除し、Google Calendar から削除する。

**前提条件**

- ユーザーがログイン状態
- 削除対象のシフトが存在

**フロー**

1. ユーザーがシフトの「削除」ボタンをクリック
2. 削除確認ダイアログ表示
3. ユーザーが「削除確認」 → DELETE /api/shifts/{id}
4. サーバー側で ShiftLessonRange を DELETE → Shift を DELETE
5. DB 削除成功 → DELETE /api/calendar-sync や Google Calendar API でイベント削除（googleEventId で識別）
6. 成功レスポンス → カレンダーへ戻る + 成功メッセージ表示

**エラーハンドリング**

- 削除取り消し（ユーザーが「いいえ」を選択）
- Google Calendar API エラー → DB は既に削除済み（スタッシュして後で手動対応）

# 5. カレンダーUI

## 5.1 概要

月カレンダーを採用。ユーザーは一目でその月のシフト状況を把握でき、日付をクリックしてシフトを登録・編集できる。

## 5.2 表示

**シフトなし**

- 通常の日付表示

**シフトあり**

- 色付き丸またはバー表示
- 色は勤務先ごとの color 属性に対応
- 複数シフトがある場合は複数の色を表示（並列 or 帯状）

**当日**

- 別途ハイライト（今日の枠線を太くするなど）

## 5.3 インタラクション

**日付クリック**

- 該当日のシフト一覧モーダル表示
- 列表: シフト情報（勤務先、開始～終了時刻）
- ボタン: 「追加」「編集」「削除」

**月ナビゲーション**

- 前月・次月へ移動

# 6. 集計機能

## 6.1 集計の概要

ユーザーが選択した期間（月、カスタム期間など）をベースに、給与・勤務時間を集計。

## 6.2 主集計

### 6.2.1 集計 BY 給与締め期間

**期間の定義例**

- 月ベース：1日 ～ 月末
- カスタム：ユーザーが「開始日」「終了日」を指定

**表示項目**
| 項目 | 説明 |
| -------------- | ----------------------------------------- |
| 総勤務時間 | 期間内の全シフトの実勤務時間合計 |
| 深夜勤務時間 | 期間内の深夜帯（nightStart ～ nightEnd）勤務時間 |
| 残業時間 | 期間内のシフトで dailyOvertimeThreshold を超過した時間 |
| 概算給与 | 期間内の全シフト給与合計 |

### 6.2.2 勤務先別内訳

期間内の勤務先ごとに以下を表示

| 項目       | 説明                   |
| ---------- | ---------------------- |
| 勤務先名   | Workplace.name         |
| 勤務先色   | Workplace.color        |
| シフト件数 | 該当期間のシフト数     |
| 勤務時間   | 勤務先ごとの総勤務時間 |
| 給与小計   | 勤務先ごとの総給与     |

## 6.3 副表示

ユーザーが期間集計を見やすくするため、以下も表示

- **前月合計**: 前月同時期の集計データ（比較用）
- **当月累計**: 1月から現在月までの累計
- **年間累計**: 1月から現在月までの累計

# 8. 給与計算仕様

## 8.1 対象・非対象

**対象**

- **NORMAL/OTHER型**: 基本時給、休日時給、深夜割増、1日所定時間外（残業）
- **LESSON型**: コマ給（perLessonWage）

**非対象（MVP では未対応）**

- 控除（税金、社会保険など）
- 週単位での残業計算
- 月60時間を超える残業（法定外残業）
- 通勤手当・その他手当

---

## 8.2 計算の基本フロー

1. 対象期間内のすべての Shift を取得
2. Shift ごとに該当する PayrollRule を時系列で確定
3. Shift ごとに以下を計算：
   - **NORMAL/OTHER型**: 実勤務時間、深夜帯勤務時間、残業時間を分類し、給与を計算
   - **LESSON型**: コマ数（endPeriod - startPeriod + 1）と perLessonWage を用いて給与を計算
4. 期間合計を集計

---

## 8.3 計算式

### 8.3.1 勤務時間の分類

**実勤務時間（H_total）**

```
H_total = (endTime - startTime - breakMinutes) / 60 [時間]
```

**深夜帯勤務時間（H_night）**

- nightStart ～ nightEnd の時間帯に該当する勤務時間
- 例：nightStart=22:00, nightEnd=05:00 の場合
  - 22:00 ～ 24:00（2時間）と 00:00 ～ 05:00（5時間）の合計
  - 翌日にまたがる場合は合算

**日中勤務時間（H_day）**

```
H_day = H_total - H_night
```

**残業時間（H_overtime）**

```
H_overtime = max(0, H_total - dailyOvertimeThreshold)
```

### 8.3.2 給与計算

**NORMAL/OTHER型のシフト（1シフト分）**

基本賃金（該当する休日判定）

- holidayType に基づいて、その日が休日か判定
- 休日 → holidayHourlyWage（NULLなら baseHourlyWage）
- 平日 → baseHourlyWage

```
日中勤務給 = H_day × baseHourlyWage

残業給 = H_overtime × baseHourlyWage × overtimeMultiplier

深夜割増給 = H_night × baseHourlyWage × (nightMultiplier - 1)
(nightMultiplier は 1.25 などの倍率。1.25 なら 25% 増)

シフト合計給 = 日中勤務給 + 残業給 + 深夜割増給
```

**LESSON型のシフト（1シフト分）**

```
コマ数 = endPeriod - startPeriod + 1

シフト合計給 = コマ数 × perLessonWage
```

**補足**

- NORMAL/OTHER型で深夜時間帯に残業がある場合は、さらに複雑だが MVP 段階では単純化
- 詳細なルールは実装時に業務ロジックで定義

---

## 8.4 具体例

### 例1：通常勤務（NORMAL シフト）

**条件**

- 勤務先：コンビニA（GENERAL）
- 日付：2026-03-20（金、平日）
- 時間：10:00 ～ 18:00
- 休憩：60分
- PayrollRule：baseHourlyWage=1100, nightStart=22:00, nightEnd=05:00, dailyOvertimeThreshold=8

**計算**

```
H_total = (18:00 - 10:00 - 60) / 60 = 7 時間
H_night = 0（深夜帯なし）
H_day = 7 時間
H_overtime = max(0, 7 - 8) = 0 時間

給与 = 7 × 1100 = 7,700 円
```

### 例2：夜勤シフト（NORMAL シフト、深夜帯含む）

**条件**

- 勤務先：別のコンビニ
- 日付：2026-03-21（土、休日判定で WEEKEND）
- 時間：22:00 ～ 翌05:00
- 休憩：30分
- PayrollRule：baseHourlyWage=1100, holidayHourlyWage=1200, nightMultiplier=1.25, nightStart=22:00, nightEnd=05:00, dailyOvertimeThreshold=8, holidayType=WEEKEND

**計算**

```
H_total = (05:00 (次日) - 22:00 - 30) / 60 = 6.5 時間
H_night = 6.5 時間（全て深夜帯）
H_day = 0 時間

日付が「土曜」なので休日判定 → holidayHourlyWage = 1200

給与 = 6.5 × 1200 × 1.25 = 9,750 円
( = 6.5 × 1200（基本） + 6.5 × 1200 × 0.25（深夜割増）)
```

### 例3：塾の授業（LESSON シフト）

**条件**

- 勤務先：塾B（CRAM_SCHOOL）
- 日付：2026-03-22（日）
- シフトタイプ：LESSON（Period 1～3、type=NORMAL）
- Timetable（通常期間）：
  - Period 1 (NORMAL): 16:30 ～ 17:30
  - Period 2 (NORMAL): 17:40 ～ 18:40
  - Period 3 (NORMAL): 18:50 ～ 19:50
- 自動計算：startTime=16:30、endTime=19:50、コマ数=3
- PayrollRule：perLessonWage=2000

**計算**

```
コマ数 = 3 - 1 + 1 = 3 コマ

給与 = 3 × 2000 = 6,000 円
```

- 保存時は勤務先作成と初期給与ルール作成を同時に行う（同時作成OFF時は勤務先のみ作成）。
- 作成確定後の遷移:
  - `GENERAL`: SCR_008（勤務先一覧）へ戻る
  - `CRAM_SCHOOL`: SCR_013（時間割新規作成）へ遷移

### SCR_011: 給与ルール作成・編集

**目的**: 勤務先ごとの給与ルールを入力・編集する。

**フォーム定義**

| フィールド   | 型     | 必須 | 仕様                                                      |
| ------------ | ------ | ---- | --------------------------------------------------------- |
| 開始日       | DATE   | ○    | YYYY-MM-DD 形式                                           |
| 終了日       | DATE   | ~    | 空白 = 現在有効; NULLの場合、開始日より後                 |
| 基本時給     | NUMBER | ◆    | GENERAL型のみ; 正の実数; 小数2位まで                      |
| コマ給       | NUMBER | ◆    | CRAM_SCHOOL型のみ; 正の実数; 小数2位まで                  |
| 休日時給     | NUMBER | ~    | GENERAL型のみ; 空白 = 基本時給と同じ                      |
| 深夜割増率   | NUMBER | ◆    | GENERAL型のみ; ≥ 1.0; 例：1.25                            |
| 残業割増率   | NUMBER | ◆    | GENERAL型のみ; ≥ 1.0; 例：1.5                             |
| 1日所定時間  | NUMBER | ◆    | GENERAL型のみ; 正の実数; 例：8.0                          |
| 深夜開始時刻 | TIME   | ◆    | GENERAL型のみ; HH:MM                                      |
| 深夜終了時刻 | TIME   | ◆    | GENERAL型のみ; HH:MM                                      |
| 休日判定     | RADIO  | ◆    | GENERAL型のみ; NONE / WEEKEND / HOLIDAY / WEEKEND_HOLIDAY |

**説明**

- **◆ 条件付き必須**: 勤務先の種類（type）に応じて表示・必須が変わる
  - **GENERAL型**: `基本時給`、`深夜割増率`、`残業割増率`、`1日所定時間`、`深夜開始時刻`、`深夜終了時刻`、`休日判定` は必須。`コマ給` は非表示
  - **CRAM_SCHOOL型**: `コマ給` は必須。その他時給関連フィールドは非表示（施設内でLESSON型のため）

### SCR_013: 塾時間割作成・編集

**目的**: 塾のコマ情報を入力・編集する。

**フォーム定義**

| フィールド | 型     | 必須 | 仕様                                                |
| ---------- | ------ | ---- | --------------------------------------------------- |
| コマ種別   | RADIO  | ○    | NORMAL(通常期) or INTENSIVE(講習期)                 |
| コマ番号   | NUMBER | ○    | 正の整数; type内で一意（e.g. 通常期1限、講習期1限） |
| 開始時刻   | TIME   | ○    | HH:MM 形式                                          |
| 終了時刻   | TIME   | ○    | HH:MM 形式; 開始時刻より後                          |

**複数コマ入力（作成時）**

- 作成画面では `コマを追加する` ボタンで入力行を増やし、複数コマを一括登録できる。
- 各行で `コマ種別 / コマ番号 / 開始時刻 / 終了時刻` を個別に入力する。
- 各行の削除が可能（最低1行は維持）。
- 同一リクエスト内で `(type, period)` の重複がある場合はエラー表示する。

### SCR_014: シフト一括登録

**目的**: 複数日のシフトを一括で登録し、Google Calendar に同期する。テンプレートパターンと異なる時間を一日ずつ調整したい場合に便利。

**ステップ1: 勤務先選択**

| フィールド | 型     | 必須 | 仕様                                          |
| ---------- | ------ | ---- | --------------------------------------------- |
| 勤務先     | SELECT | ○    | 登録済み勤務先から選択; デフォルト = 前回選択 |

**ステップ2: カレンダー表示と日付選択**

- **表示**: 月間カレンダー（SCR_003 と異なる専用UI）
- **日付選択**: 複数日をクリックして選択（選択状態を背景色で表示）
- **選択日数表示**: 「選択中: N日」と表示
- **クリア機能**: 「選択をリセット」ボタンで全日付選択をクリア

**ステップ3: デフォルト値設定と選択日の詳細入力**

**3-1: デフォルト値設定（上部・固定）**

頻繁に使う設定値をあらかじめ定義し、選択日に一括適用できます。

| フィールド             | 型     | 必須 | 仕様                                          |
| ---------------------- | ------ | ---- | --------------------------------------------- |
| デフォルトシフトタイプ | RADIO  | ○    | NORMAL / LESSON / OTHER; 適用時のプリセット値 |
| デフォルト開始時刻     | TIME   | ◆    | NORMAL/OTHER型のみ表示; HH:MM 形式            |
| デフォルト終了時刻     | TIME   | ◆    | NORMAL/OTHER型のみ表示; HH:MM 形式            |
| デフォルト休憩時間     | NUMBER | ○    | 分単位; 0 ～ 240                              |
| デフォルトコマ種別     | RADIO  | ◆    | LESSON型のみ表示; NORMAL / INTENSIVE          |
| デフォルトコマ範囲     | SELECT | ◆    | LESSON型のみ表示; Period 1～N                 |

**アクション**

| ボタン             | 動作                                   |
| ------------------ | -------------------------------------- |
| デフォルト値を適用 | 下記の全選択日に上記の設定値を一括入力 |
| リセット           | デフォルト値の入力欄をクリア           |

**3-2: 選択日の詳細入力（スクロール対応、常に展開）**

選択した各日付について、スクロール可能な縦長レイアウトで、常時展開された状態で入力項目を表示：

| フィールド   | 型     | 必須 | 仕様                                                          |
| ------------ | ------ | ---- | ------------------------------------------------------------- |
| 日付表示     | TEXT   | ー   | 「2026年3月5日（木）」など読み取り専用                        |
| シフトタイプ | RADIO  | ○    | NORMAL / LESSON / OTHER; デフォルト = NORMAL                  |
| 開始時刻     | TIME   | ◆    | NORMAL/OTHER型のみ表示; HH:MM 形式                            |
| 終了時刻     | TIME   | ◆    | NORMAL/OTHER型のみ表示; HH:MM 形式; 開始時刻より後            |
| 休憩時間     | NUMBER | ○    | 分単位; デフォルト = 0; 0 ～ 240                              |
| コマ種別     | RADIO  | ◆    | LESSON型のみ表示; NORMAL（通常期）/ INTENSIVE（講習期）       |
| コマ範囲     | SELECT | ◆    | LESSON型のみ表示; Period 1～N; startPeriod ≤ endPeriod を確認 |

**UI実装の詳細**

- **レイアウト**: 各日付を独立したカード/セクション形式で縦並びに表示（チェックボックス + 日付 + 入力フィールド）
- **スクロール**: 複数日同時登録時は、選択日の数に応じてスクロール可能な高さ制限を設ける（例：最大高さ 400px～600px）
- **常時展開**: 完全に展開された状態で表示され、たためる機能は不要
- **デフォルト値の有効性**: デフォルト値を適用した後でも、各日付ごとに個別に値を編集可能

**ステップ4: 確定**

| ボタン     | 動作                            |
| ---------- | ------------------------------- |
| 登録       | 全選択日のシフトをDB登録 + 同期 |
| キャンセル | SCR_003 (カレンダー) へ戻る     |

**バリデーション**

1. **選択日必須**: 1日以上選択されているか確認
2. **時間バリデーション**: NORMAL/OTHER型で開始時刻 < 終了時刻
3. **条件付き必須**: シフトタイプに応じて必須フィールド確認
4. **コマ範囲**: LESSON型選択時、有効なコマが存在するか確認

**エラーメッセージ**

| エラーコード | メッセージ                             |
| ------------ | -------------------------------------- |
| ERR_001      | 〇〇は必須項目です                     |
| ERR_002      | 開始時刻は終了時刻より前にしてください |
| ERR_003      | 1日以上選択してください                |
| ERR_004      | 塾の授業は時間割が登録されていません   |

### SCR_015: シフト確定

**目的**: 本日までの未確定シフトを確認・修正し、確定処理を行う。上部では未確定シフトをカードリスト形式で表示（時間変更可能）、下部では今月の確定済みシフトを勤務先ごとにリストアップ。

**画面構成**

**Part 1: 未確定シフトリスト（上部）**

**リスト条件**

- 日付が本日以前（today 以前）
- `isConfirmed = false`

**リスト順序**

- `date ASC`（古い日付から新しい日付へ）

**リストが空の場合**

- 「未確定シフトはありません」メッセージを表示

**カード表示**

各未確定シフトを以下の形式でカード表示。複数カードが存在する場合はスクロール可能な領域に縦並びで表示：

| 要素       | 内容                                                 |
| ---------- | ---------------------------------------------------- |
| 日付       | 「2026年3月5日（木）」など                           |
| 勤務先名   | 勤務先の名称                                         |
| 開始時刻   | **編集可能な入力フィールド** (TIME フォーマット)     |
| 終了時刻   | **編集可能な入力フィールド** (TIME フォーマット)     |
| 休憩時間   | **編集可能な入力フィールド** (数値、分単位)          |
| 確定ボタン | シフト確定処理トリガー                               |
| 削除ボタン | シフト削除処理トリガー（ユーザー確認ダイアログ表示） |

**インタラクション**

| アクション                 | 動作                                                          |
| -------------------------- | ------------------------------------------------------------- |
| **時刻/休憩時間編集**      | 入力値がリアルタイムに更新される。確定時に新値でDB更新される  |
| **確定ボタンクリック**     | シフトの `isConfirmed` を `true` に更新 → DB反映 → カード削除 |
| **削除ボタンクリック**     | 削除確認ダイアログ表示 → 確認後にシフト削除 → カード削除      |
| **編集値のバリデーション** | 開始時刻 < 終了時刻、休憩時間 ≥ 0 を確認                      |

**Part 2: 今月の確定済みシフトリスト（下部）**

**リスト条件**

- 当月内（例：2026年3月1日～2026年3月31日）
- `isConfirmed = true`

**リスト順序**

- 勤務先ごとにグループ化
- グループ内は日付 ASC

**表示形式**

勤務先ごとにセクション分けし、各セクション内で以下の情報をテーブル形式で表示：

| 列名     | 内容                            |
| -------- | ------------------------------- |
| 日付     | 「2026年3月5日」                |
| 時間帯   | 「10:00 ～ 18:00（実働7.5h）」  |
| 確定済み | チェックマーク（✓）またはバッジ |

**セクション表示例**

```
◆ コンビニA
┌─────────────┬────────────────┬──────────┐
│ 日付        │ 時間帯          │ 状態     │
├─────────────┼────────────────┼──────────┤
│ 2026年3月5日 │ 10:00 ～ 18:00 │ ✓        │
│ 2026年3月12日│ 10:00 ～ 18:00 │ ✓        │
└─────────────┴────────────────┴──────────┘

◆ 英語塾
┌─────────────┬────────────────┬──────────┐
│ 日付        │ 時間帯          │ 状態     │
├─────────────┼────────────────┼──────────┤
│ 2026年3月6日 │ 16:30 ～ 19:50 │ ✓        │
└─────────────┴────────────────┴──────────┘
```

**確定済みシフトが空の場合**

- セクション全体を非表示（「今月の確定済みシフトはありません」といったメッセージは不要）

**バリデーション（Part 1 の編集値）**

1. **時刻バリデーション**: 開始時刻 < 終了時刻
2. **休憩時間**: 0 ≤ breakMinutes ≤ 240
3. **編集後の確定**: 確定ボタン押下時に上記を再確認

**バリデーションエラーメッセージ**

| エラーコード | メッセージ                                 |
| ------------ | ------------------------------------------ |
| ERR_VAL_003  | 開始時刻は終了時刻より前にしてください     |
| ERR_VAL_004  | 休憩時間は0～240分の範囲で入力してください |
| ERR_SYS_001  | シフト確定に失敗しました。もう一度お試し   |

**ページ全体の背景・レイアウト**

- **背景**: ページ背景はダッシュボームと同色
- **Part 1 / Part 2 の区切り**: 視覚的に区別できるセパレーターを配置
- **レスポンシブ**: モバイル表示時、カード幅を制約し読みやすさを維持

# 10. バリデーション・エラーハンドリング

## 10.1 バリデーションエラー（クライアント側）

| エラーコード | パターン       | ユーザー表示                                 | 処理                       |
| ------------ | -------------- | -------------------------------------------- | -------------------------- |
| ERR_VAL_001  | 必須項目が空   | 「〇〇は必須項目です」                       | フォーカス移動 + 赤背景    |
| ERR_VAL_002  | メール形式不正 | 「正しいメールアドレスを入力してください」   | 入力値削除，フォーカス移動 |
| ERR_VAL_003  | 時刻が不正     | 「開始時刻は終了時刻より前にしてください」   | 入力値ハイライト           |
| ERR_VAL_004  | 数値範囲外     | 「〇〇は〇〇以上〇〇以下で入力してください」 | 入力値削除                 |

## 10.2 システムエラー（サーバー側）

| エラーコード | 原因                       | ユーザー表示                                   | 対応                        |
| ------------ | -------------------------- | ---------------------------------------------- | --------------------------- |
| SYS_001      | DB接続失敗                 | 「通信に失敗しました。もう一度お試しください」 | リトライボタン表示          |
| SYS_002      | Google Calendar API エラー | 「Googleカレンダーとの同期に失敗しました」     | ログ記録 + 部分成功状態記録 |
| SYS_003      | データ競合                 | 「別ユーザーが同時に更新しました」             | 画面リロード促し            |

## 10.3 エラーメッセージの表示

- **Must-fix エラー**: 赤トースト or インラインエラー表示
- **警告**: 黄トースト表示（上書き許容）
- **成功**: 緑トースト表示 → 2秒後に自動非表示

# 11. 同期・セキュリティ・レスポンシブ

## 11.1 Google Calendar 同期フロー

| シフト操作 | 同期アクション | イベント属性                                           | 注記                 |
| ---------- | -------------- | ------------------------------------------------------ | -------------------- |
| **作成**   | イベント作成   | title: "[NORMAL] 勤務先名" or "[LESSON] 勤務先名" etc. | googleEventId を保存 |
| **編集**   | イベント更新   | 時間・タイトルを更新（色は Google Calendar 既定色）    | googleEventId で識別 |
| **削除**   | イベント削除   | googleEventId にて削除                                 | DB削除と同期         |

**同期失敗時の処理**

- DB には既に保存済み
- ユーザーに「Googleカレンダーとの同期に失敗しました」と表示
- 詳細ログはサーバー側に記録
- 将来「同期再試行」機能で手動対応
