# 1. システム概要

本システムは  
**個人用シフト・給与概算管理Webアプリ**である。

主な機能

- シフト登録・編集・削除
- 月間カレンダー表示
- 概算給与計算・集計
- Google Calendar同期

# 2. システム構成

**アーキテクチャレイヤー**

| レイヤー           | 技術スタック           | 説明                                     |
| ------------------ | ---------------------- | ---------------------------------------- |
| **フロントエンド** | Browser + React 19     | ユーザーブラウザで実行される UI レイヤー |
| **バックエンド**   | Next.js App Router     | API ルートと Server Components           |
| **ORM層**          | Prisma + Neon Adapter  | データベース抽象化レイヤー               |
| **データベース**   | Neon（PostgreSQL互換） | サーバーレス PostgreSQL データベース     |
| **外部連携**       | Google Calendar API    | カレンダー同期機能                       |

**認証**

- NextAuth.js を使用したメール認証
- 単一ユーザー対応

**通信フロー**

Browser (React UI) → Next.js Routes → Prisma ORM → Neon DB

外部連携: Neon ← → Google Calendar API

# 3. ドメインモデル

本システムは単一ユーザー専用である。各ドメインモデルの関連性は以下の通り。

**エンティティ関連性**

- **User** (ユーザー)
  - **Workplace** (勤務先) - 複数登録可能
    - **PayrollRule** (給与ルール) - 複数、時系列管理
    - **Timetable** (時間割) - 複数、塾のみ
    - **Shift** (シフト) - 複数
      - **ShiftLessonRange** (授業コマ範囲) - 複数、LESSON タイプのシフトのみ

---

## 3.1 User

ログインユーザー

**説明**: NextAuth.js でログイン管理されるユーザー。本システムは単一ユーザー対応のため、通常1件のみ存在する。

| 属性      | 型       | 説明                         |
| --------- | -------- | ---------------------------- |
| id        | UUID     | ユーザーの一意識別子         |
| email     | string   | メールアドレス（ログインID） |
| createdAt | datetime | アカウント作成日時           |

**制約条件**

- email は一意（unique）
- email は有効なメール形式

**使用例**

- email: user@example.com
- createdAt: 2026-01-15T08:30:00Z

---

## 3.2 Workplace

勤務先（複数登録可能）

**説明**: ユーザーが勤務する企業・塾など。シフトの根拠となる組織。

| 属性      | 型       | 説明                                       |
| --------- | -------- | ------------------------------------------ |
| id        | UUID     | 勤務先の一意識別子                         |
| name      | string   | 勤務先名（例：「コンビニA」「塾B」）       |
| color     | string   | UIに表示する色（HEX形式）                  |
| type      | enum     | 勤務先の種別（GENERAL または CRAM_SCHOOL） |
| createdAt | datetime | 勤務先登録日時                             |

**制約条件**

- name は空でない
- color は HEX カラーコード（#RRGGBB）形式
- 1ユーザーあたり複数勤務先可能
- 削除時は関連する Shift, PayrollRule も削除される（CASCADE）

**使用例**

- name: "コンビニA"、type: GENERAL、color: "#FF5733"
- name: "英語塾"、type: CRAM_SCHOOL、color: "#3366FF"

**type の説明**

- **GENERAL**: 通常の勤務先（コンビニ、飲食店など）
- **CRAM_SCHOOL**: 塾（Timetable と ShiftLessonRange を使用）

---

## 3.3 PayrollRule

給与ルール履歴（勤務先ごとの時給管理）

**説明**: 勤務先ごとの時給・割増ルールを時系列管理。同一勤務先で複数ルールが存在し、シフト日に該当する有効なルールを自動選択する。

**ルール選択ロジック**: `startDate ≤ shift.date < endDate` を満たすルール中、`startDate` が最新（最大）のものを選択。同一勤務先内では適用期間の重複を許さない。

| 属性                   | 型      | 説明                                          |
| ---------------------- | ------- | --------------------------------------------- |
| id                     | UUID    | ルールの一意識別子                            |
| workplaceId            | UUID    | 勤務先ID（外部キー）                          |
| startDate              | date    | ルール適用開始日                              |
| endDate                | date?   | ルール適用終了日（NULLの場合は現在有効）      |
| baseHourlyWage         | number  | 基本時給（円、小数2位まで。NORMAL/OTHER型用） |
| perLessonWage          | number? | コマ給（円、小数2位まで。LESSON型用）         |
| holidayAllowanceHourly | number  | 休日手当（円/時の加算手当。デフォルト0）      |
| nightPremiumRate       | number  | 深夜割増率（例：0.25 = 25%）                  |
| dailyOvertimeThreshold | number  | 1日の所定時間（超過分が残業対象）             |
| overtimePremiumRate    | number  | 残業割増率（将来拡張。例：0.25 = 25%）        |
| holidayType            | enum    | 休日判定タイプ                                |

**制約条件**

- startDate と endDate が重複しない（同一勤務先内で）
- endDate は NULLか startDate より後
- baseHourlyWage > 0
- perLessonWage > 0
- nightPremiumRate ≥ 0
- overtimePremiumRate ≥ 0
- dailyOvertimeThreshold > 0（時間、実数可）

**holidayType の説明**

- **NONE**: 休日判定なし（毎日同じ時給）
- **WEEKEND**: 土日を休日扱い
- **HOLIDAY**: 指定の祝日のみ休日扱い（将来拡張）
- **WEEKEND_HOLIDAY**: 土日と祝日を休日扱い

**使用例**

勤務先「コンビニA」:

- Rule 1: 2026-01-01 ～ 2026-03-31、baseHourlyWage=1100、nightPremiumRate=0.25
- Rule 2: 2026-04-01 ～ NULL、baseHourlyWage=1150、nightPremiumRate=0.25

勤務先「塾B」（CRAM_SCHOOL）:

- Rule 1: 2026-01-01 ～ 2026-03-31、perLessonWage=2000、nightPremiumRate=0.0
- Rule 2: 2026-04-01 ～ NULL、perLessonWage=2500、nightPremiumRate=0.0

## 3.4 Timetable

塾の時間割（CRAM_SCHOOL タイプの勤務先のみ使用）

**説明**: 塾の授業コマごとの開始・終了時刻を定義。通常期間と講習期間など複数のセットを管理可能。シフト登録（LESSON型）時に、選択したコマ範囲から`startTime/endTime`を自動計算する際に使用。

**用途**：

- LESSON型シフト時：「5限～7限」のように選択すると、該当コマの時刻を自動取得
- NORMAL型の場合も塾に登録可能（手入力、会議や事務作業など授業以外の業務を記録時に使用）

| 属性        | 型   | 説明                                                                    |
| ----------- | ---- | ----------------------------------------------------------------------- |
| id          | UUID | 時間割の一意識別子                                                      |
| workplaceId | UUID | 勤務先ID（CRAM_SCHOOL のみ、外部キー）                                  |
| type        | enum | 授業タイプ（NORMAL または INTENSIVE）                                   |
| period      | int  | コマ番号（1, 2, 3, ... など。typeごとに独立、例：通常期1限、講習期1限） |
| startTime   | time | コマの開始時刻（HH:MM）                                                 |
| endTime     | time | コマの終了時刻（HH:MM）                                                 |

**制約条件**

- workplaceId は type が CRAM_SCHOOL
- (workplaceId, type, period) は一意（type ごとに独立した period 番号体系を持つ）
- startTime と endTime は同時刻不可
- endTime < startTime の場合、翌日終了のコマとして扱う（例：23:50 ～ 翌00:30）

**type の説明**

- **NORMAL**: 通常期間の授業
- **INTENSIVE**: 講習期間（集中講座・特別授業等）

同一勤務先・同一period内で、両typeのコマを登録可能（時期に応じて使い分け）。

**使用例**

塾「英語塾」の時間割（通常期と講習期で時間帯が異なる場合の例）:

通常期（type=NORMAL）:

- Period 1: 16:30 ～ 17:30
- Period 2: 17:40 ～ 18:40
- Period 3: 18:50 ～ 19:50

講習期（type=INTENSIVE）:

- Period 1: 15:00 ～ 16:20
- Period 2: 16:30 ～ 17:50
- Period 3: 18:00 ～ 19:20

---

## 3.5 Shift

シフト（実際の勤務）

**説明**: ユーザーが登録する実際の勤務シフト。勤務先ごと・日ごとに複数登録可能。Google Calendar と同期。

**LESSON型の特徴**：

- ユーザーは「5限～7限」のようにコマ範囲を選択
- Timetableから該当コマの時刻を自動計算 → `startTime/endTime` に格納
- ShiftLessonRange で「どのコマ範囲か」を記録
- Google Calendar には自動計算された時刻で登録される

**NORMAL型（塾での会議・事務作業など授業以外の業務）の特徴**

- ユーザーが時刻を手入力
- 時給で給与計算される（PayrollRule の baseHourlyWage を使用）

| 属性          | 型       | 説明                                             |
| ------------- | -------- | ------------------------------------------------ |
| id            | UUID     | シフトの一意識別子                               |
| workplaceId   | UUID     | 勤務先ID（外部キー）                             |
| date          | date     | シフトの勤務開始日                               |
| startTime     | time     | シフト開始時刻（HH:MM）                          |
| endTime       | time     | シフト終了時刻（HH:MM）                          |
| breakMinutes  | int      | 休憩時間（分単位、デフォルト0）                  |
| shiftType     | enum     | シフトのタイプ（NORMAL, LESSON, OTHER）          |
| comment       | string?  | 表示・Google Calendar イベント名用の任意コメント |
| isConfirmed   | boolean  | シフト確定フラグ（デフォルト false）             |
| googleEventId | string?  | Google Calendar のイベントID（同期用）           |
| createdAt     | datetime | シフト登録日時                                   |

**制約条件**

- 同一勤務先の時間帯重複は、日跨ぎ解釈後の開始日時・終了日時で判定する（バリデーション段階で警告）
- startTime と endTime は同時刻不可
- endTime < startTime の場合、翌日終了のシフトとして扱う
- breakMinutes ≥ 0
- comment は任意。未入力・空文字・空白のみは NULL として保存
- comment は最大100文字、改行不可
- shiftType = LESSON の場合、ShiftLessonRange が1件以上存在
- shiftType = LESSON 以外の場合、ShiftLessonRange は存在しない

**勤務先表示名・イベント名**

- comment が NULL の場合: `勤務先名`
- comment がある場合: `勤務先名 (コメント)`
- 従来の塾タイプ NORMAL シフトに対する自動の ` (事務)` 表記は廃止する。事務として表示したい場合は comment に `事務` を入力する。

**shiftType の説明**

- **NORMAL**: 通常勤務（手入力。勤務先がCRAM_SCHOOLの場合は会議や事務作業など授業以外の業務も含む。時給で給与計算）
- **LESSON**: 塾の授業（CRAM_SCHOOL のみ。ShiftLessonRange で「何コマ目から何コマ目まで」を定義。時刻は Timetable から自動計算）
- **OTHER**: その他（自由入力）

**使用例**

- 2026-03-20、コンビニA、10:00 ～ 18:00、NORMAL、breakMinutes=60
- 2026-03-21、塾B、16:30 ～ 19:50、LESSON（Period 1～3）、breakMinutes=0

## 3.6 ShiftLessonRange

塾のコマ範囲（Shift.shiftType = LESSON の場合のみ）

**説明**: LESSON タイプのシフトが どのコマから どのコマまでをカバーするかを記録。Timetable のコマに対応する。複数コマの授業を1シフトで記録可能。

**注記**：`startPeriod ≤ endPeriod` の同じ `type`（NORMAL or INTENSIVE）のコマ範囲を指す。異なる `type` のコマを混在させることはできない。

| 属性        | 型   | 説明                                         |
| ----------- | ---- | -------------------------------------------- |
| id          | UUID | レコードの一意識別子                         |
| shiftId     | UUID | シフトID（外部キー）                         |
| startPeriod | int  | 開始コマ番号                                 |
| endPeriod   | int  | 終了コマ番号（startPeriodと同じ場合は1コマ） |

**制約条件**

- shiftId の Shift.shiftType = LESSON
- startPeriod ≤ endPeriod
- startPeriod, endPeriod は Timetable のコマの period に存在し、かつ同一 `type`（NORMAL or INTENSIVE）
- 1シフトあたり1件（複数コマの授業を1つのシフトで表現）

**使用例**

- Shift（2026-03-21、塾B、16:30 ～ 19:50、LESSON）
  → ShiftLessonRange: startPeriod=1, endPeriod=3

# 4. 主要ユースケース

---

## UC-001: シフト登録

**目的**: 新しいシフトをDB に登録し、Google Calendar に同期する。

**前提条件**

- ユーザーがログイン状態
- 勤務先が最低1件登録済み
- （LESSON の場合）塾の時間割が設定済み

**フロー**

1. ユーザーがカレンダーから日付をクリック
2. シフト入力フォーム表示（勤務先、シフトタイプなど）
3. ユーザーが情報を入力
   - **NORMAL型の場合**：開始時刻・終了時刻を手入力
   - **LESSON型の場合**：コマ範囲（e.g. Period 1～3）を選択 → サーバー側で Timetable から時刻を自動計算
4. クライアント側でバリデーション（必須項目、時刻の妥当性など）
5. NORMAL/OTHER型で終了時刻 < 開始時刻の場合、翌日終了として扱う確認ダイアログを表示
6. バリデーション OK → POST /api/shifts（Shift + ShiftLessonRange を作成）
7. サーバー側で：
   - LESSON型の場合、Timetable から `startTime/endTime` を自動計算
   - Shift を INSERT → ShiftLessonRange を INSERT
8. DB 保存成功 → POST /api/calendar-sync にイベント作成を依頼（自動計算した時刻を使用）
9. Google Calendar API でイベント作成 → googleEventId を Shift に UPDATE
10. 成功レスポンス → 前画面へ戻る + 成功メッセージ表示

**エラーハンドリング**

- バリデーションエラー → UI でユーザーに指摘し、修正を促す
- Google Calendar API エラー → DB には保存済み、スタッシュして後で再同期（将来）

---

## UC-002: シフト編集

**目的**: 既存シフトを編集し、Google Calendar に反映する。

**前提条件**

- ユーザーがログイン状態
- 編集対象のシフトが存在

**フロー**

1. ユーザーがカレンダーまたはシフト一覧からシフトを選択
2. シフト詳細/編集フォーム表示（既存値をプリセット）
3. ユーザーが値を変更 → 「更新」ボタンをクリック
4. クライアント側でバリデーション
5. 更新後に終了時刻 < 開始時刻となる場合、翌日終了として扱う確認ダイアログを表示
6. バリデーション OK → PUT /api/shifts/{id}（Shift + ShiftLessonRange を更新）
7. サーバー側で Shift を UPDATE + ShiftLessonRange を DELETE/INSERT
8. DB 更新成功 → PUT /api/calendar-sync/{googleEventId} でイベント更新
9. Google Calendar API でイベント更新
10. 成功レスポンス → 前画面へ戻る + 成功メッセージ表示

**エラーハンドリング**

- 同じシフト内での時間帯重複チェック（警告のみ）
- Google Calendar API エラー → DB は更新済み

---

## UC-003: シフト削除

**目的**: シフトを削除し、Google Calendar から削除する。

**前提条件**

- ユーザーがログイン状態
- 削除対象のシフトが存在

**フロー**

1. ユーザーがシフトの「削除」ボタンをクリック
2. 削除確認ダイアログ表示
3. ユーザーが「削除確認」 → DELETE /api/shifts/{id}
4. サーバー側で ShiftLessonRange を DELETE → Shift を DELETE
5. DB 削除成功 → DELETE /api/calendar-sync や Google Calendar API でイベント削除（googleEventId で識別）
6. 成功レスポンス → カレンダーへ戻る + 成功メッセージ表示

**エラーハンドリング**

- 削除取り消し（ユーザーが「いいえ」を選択）
- Google Calendar API エラー → DB は既に削除済み（スタッシュして後で手動対応）

# 5. カレンダーUI

## 5.1 概要

月カレンダーを採用。ユーザーは一目でその月のシフト状況を把握でき、日付をクリックしてシフトを登録・編集できる。

## 5.2 表示

**シフトなし**

- 通常の日付表示

**シフトあり**

- 色付き丸またはバー表示
- 色は勤務先ごとの color 属性に対応
- 複数シフトがある場合は複数の色を表示（並列 or 帯状）

**当日**

- 別途ハイライト（今日の枠線を太くするなど）

## 5.3 インタラクション

**日付クリック**

- 該当日のシフト一覧モーダル表示
- 列表: シフト情報（勤務先、開始～終了時刻）
- ボタン: 「追加」「編集」「削除」

**月ナビゲーション**

- 前月・次月へ移動

# 6. 集計機能

## 6.1 集計の概要

ユーザーが選択した期間（月、カスタム期間など）をベースに、給与・勤務時間を集計。

## 6.2 主集計

### 6.2.1 集計 BY 給与締め期間

**期間の定義例**

- 月ベース：1日 ～ 月末
- カスタム：ユーザーが「開始日」「終了日」を指定

**表示項目**
| 項目 | 説明 |
| -------------- | ----------------------------------------- |
| 総勤務時間 | 期間内の全シフトの実勤務時間合計 |
| 深夜勤務時間 | 期間内の深夜帯（22:00 ～ 05:00）勤務時間 |
| 残業時間 | 期間内のシフトで dailyOvertimeThreshold を超過した時間（将来拡張の参考値） |
| 概算給与 | 期間内の全シフト給与合計 |

### 6.2.2 勤務先別内訳

期間内の勤務先ごとに以下を表示

| 項目       | 説明                   |
| ---------- | ---------------------- |
| 勤務先名   | Workplace.name         |
| 勤務先色   | Workplace.color        |
| シフト件数 | 該当期間のシフト数     |
| 勤務時間   | 勤務先ごとの総勤務時間 |
| 給与小計   | 勤務先ごとの総給与     |

## 6.3 副表示

ユーザーが期間集計を見やすくするため、以下も表示

- **前月合計**: 前月同時期の集計データ（比較用）
- **当月累計**: 1月から現在月までの累計
- **年間累計**: 1月から現在月までの累計

# 7. Google Calendar同期

## 7.1 基本ルール

| 操作     | 処理                                        | 詳細                                                   |
| -------- | ------------------------------------------- | ------------------------------------------------------ |
| 作成     | イベント作成                                | DB保存後、その日のイベント ID を shifts テーブルに記録 |
| 編集     | イベント更新                                | 既存の googleEventId でイベント再作成または更新        |
| 削除     | イベント削除                                | googleEventId をキーに削除                             |
| 同期失敗 | DB には保存済み、同期ステータスをスタッシュ | 将来の「同期再試行」機能で対応予定                     |

## 7.2 Google Calendar 側の編集について

Google Calendar 側で シフトのイベントを編集しても、アプリには反映しない。データの正はアプリケーション DB。

**理由**: 複数勤務先・複数デバイスでのシフト管理の一貫性を保つため、DB を単一の真実の根拠地（SSOT）とする。

## 7.3 イベント属性のマッピング

| Shift 属性               | Google Calendar イベント属性             | 説明                                                                      |
| ------------------------ | ---------------------------------------- | ------------------------------------------------------------------------- |
| date, startTime, endTime | start, end                               | ISO 8601 形式で保存。endTime < startTime の場合、end は翌日日時として保存 |
| workplaceId              | カスタムプロパティ（extendedProperties） | JSONで勤務先ID等を保存（検索用）                                          |
| workplace.name, comment  | title                                    | comment がある場合は `勤務先名 (コメント)`、ない場合は `勤務先名` を使用  |
| color（勤務先）          | （指定しない）                           | colorId は送信せず、Google Calendar 側の既定色を使用                      |

# 8. 給与計算仕様

## 8.1 対象・非対象

**対象**

- **NORMAL/OTHER型**: 基本給、休日手当（時間あたり加算）、深夜割増
- **LESSON型**: コマ給（perLessonWage）

**非対象（MVP では未対応）**

- 控除（税金、社会保険など）
- 週単位での残業計算
- 月60時間を超える残業（法定外残業）
- 通勤手当・その他手当

---

## 8.2 計算の基本フロー

1. 対象期間内のすべての Shift を取得
2. Shift ごとに該当する PayrollRule を時系列で確定
3. Shift ごとに以下を計算：
   - **NORMAL/OTHER型**: 実勤務時間、深夜帯勤務時間、休日時間を分類し、給与を計算
   - **LESSON型**: コマ数（endPeriod - startPeriod + 1）と perLessonWage を用いて給与を計算
4. 期間合計を集計

---

## 8.3 計算式

### 8.3.1 勤務時間の分類

**実勤務時間（H_total）**

```
H_total = (endTime - startTime - breakMinutes) / 60 [時間]
```

- endTime < startTime の場合、endTime は翌日の時刻として計算する。

**深夜帯勤務時間（H_night）**

- 22:00 ～ 05:00 の時間帯に該当する勤務時間（固定）
- 例：22:00 ～ 05:00 の場合
  - 22:00 ～ 24:00（2時間）と 00:00 ～ 05:00（5時間）の合計
  - 翌日にまたがる場合は合算

**基本時間（H_base）**

```
H_base = H_total - H_night
```

**休日時間（H_holiday）**

```
H_holiday = 休日判定に合致する日の場合 H_total、それ以外は 0
```

### 8.3.2 給与計算

**NORMAL/OTHER型のシフト（1シフト分）**

基本前提

- 休日判定は `holidayType` に基づく
- 深夜割増率は `nightPremiumRate`（0以上）
- 休日手当は `holidayAllowanceHourly`（円/時、0以上）

```
基本支給 = H_base × baseHourlyWage

深夜支給 = H_night × baseHourlyWage × (1 + nightPremiumRate)

休日支給 = H_holiday × holidayAllowanceHourly

シフト合計給 = round(基本支給) + round(深夜支給) + round(休日支給)
```

※ `overtimePremiumRate` は将来拡張用に保持するが、現時点のV2計算では使用しない。

**LESSON型のシフト（1シフト分）**

```
コマ数 = endPeriod - startPeriod + 1

シフト合計給 = コマ数 × perLessonWage
```

**補足**

- 休日時間は基本時間・深夜時間とは独立に扱う（休日夜勤では深夜支給と休日支給の両方が発生）
- 詳細なルールは実装時に業務ロジックで定義

---

## 8.4 具体例

### 例1：通常勤務（NORMAL シフト）

**条件**

- 勤務先：コンビニA（GENERAL）
- 日付：2026-03-20（金、平日）
- 時間：10:00 ～ 18:00
- 休憩：60分
- PayrollRule：baseHourlyWage=1100, nightPremiumRate=0.25, holidayAllowanceHourly=0

**計算**

```
H_total = (18:00 - 10:00 - 60) / 60 = 7 時間
H_night = 0（深夜帯なし）
H_base = 7 時間
H_holiday = 0 時間

基本支給 = 7 × 1100 = 7,700 円
深夜支給 = 0 円
休日支給 = 0 円

給与 = 7,700 円
```

### 例2：夜勤シフト（NORMAL シフト、深夜帯含む）

**条件**

- 勤務先：別のコンビニ
- 日付：2026-03-21（土、休日判定で WEEKEND）
- 時間：22:00 ～ 翌05:00
- 休憩：30分
- PayrollRule：baseHourlyWage=1100, holidayAllowanceHourly=100, nightPremiumRate=0.25, holidayType=WEEKEND

**計算**

```
H_total = (05:00 (次日) - 22:00 - 30) / 60 = 6.5 時間
H_night = 6.5 時間（全て深夜帯）
H_base = 0 時間
H_holiday = 6.5 時間

基本支給 = 0 × 1100 = 0 円
深夜支給 = 6.5 × 1100 × 1.25 = 8,937.5 円 → 8,938 円
休日支給 = 6.5 × 100 = 650 円

給与 = 8,938 + 650 = 9,588 円
```

### 例3：塾の授業（LESSON シフト）

**条件**

- 勤務先：塾B（CRAM_SCHOOL）
- 日付：2026-03-22（日）
- シフトタイプ：LESSON（Period 1～3、type=NORMAL）
- Timetable（通常期間）：
  - Period 1 (NORMAL): 16:30 ～ 17:30
  - Period 2 (NORMAL): 17:40 ～ 18:40
  - Period 3 (NORMAL): 18:50 ～ 19:50
- 自動計算：startTime=16:30、endTime=19:50、コマ数=3
- PayrollRule：perLessonWage=2000

**計算**

```
コマ数 = 3 - 1 + 1 = 3 コマ

給与 = 3 × 2000 = 6,000 円
```

### SCR_003: カレンダー

**目的**: 月間シフト一覧を視覚的に表示し、シフト登録・編集・削除のエントリーポイントを提供する。

**ヘッダー機能**

| 要素       | 仕様                                                     |
| ---------- | -------------------------------------------------------- |
| 月選択     | 前月・次月ボタン、または月ピッカー                       |
| 表示月     | 「2026年3月」など YYYY年M月 形式で表示                   |
| 当日マーク | 本日の日付の背後に薄めの塗りつぶし丸を配置（ハイライト） |

**カレンダー表示**

- **形式**: 月間カレンダー（日曜始まり）
- **日付セル**: 日付の数字を表示
- **空きセル**: 前月・次月の日付は薄く表示（クリック不可）

**シフト表示（新規仕様）**

​日付セルの下部に、その日に登録されているシフト数に応じて色付き丸を表示：

- **1件のシフト**: そのシフトの勤務先の色の丸（直径約6～8px）を1つ表示
- **複数件**: 各シフトの勤務先色に対応した丸を**横並び**に表示（最大5～6件、超過は「+N」テキスト）
- **色の由来**: 各丸の色は `Workplace.color` フィールドから取得
- **配置**: 日付の数字より下、セルの中央下部に配置

**例**

```
┌─────────────┐
│      5      │   ← 日付数字
│             │
│    ● ●     │   ← シフトが2件（勤務先Aとワークプレイス Bの色の丸）
└─────────────┘

┌─────────────┐
│      12     │
│             │
│  ● ● ● +2  │   ← シフトが5件以上（色の丸3つ + "+2" テキスト）
└─────────────┘
```

**インタラクション**

| アクション           | 動作                                                                                            |
| -------------------- | ----------------------------------------------------------------------------------------------- |
| **日付クリック**     | シフトがない場合 → SCR_004 (シフト入力) へ遷移                                                  |
| **シフト丸クリック** | その日のシフト一覧モーダルを表示 → シフト選択で編集/削除                                        |
| **マウスホバー**     | 丸上に勤務先表示名をツールチップ表示（2秒後に表示）。comment がある場合は `勤務先名 (コメント)` |

**シフト一覧モーダル**（日付クリック時の選択肢）

- **表示内容**: その日のシフト一覧（時刻、勤務先表示名、給与予想含む）。comment がある場合、勤務先表示名は `勤務先名 (コメント)`
- **アクション**:
  - シフト行クリック → SCR_005 (シフト編集) へ遷移
  - 「新規追加」ボタン → SCR_004 (シフト入力) へ遷移
  - 「削除」アイコン → SCR_006 (削除確認) へ遷移

### SCR_004: シフト入力

**目的**: 新規シフトをDB に登録し、Google Calendar に同期する。

**フォーム定義**

| フィールド   | 型     | 必須 | 仕様                                                                                                 |
| ------------ | ------ | ---- | ---------------------------------------------------------------------------------------------------- |
| 勤務先       | SELECT | ○    | 登録済み勤務先から選択; デフォルト = 前回選択した勤務先                                              |
| 日付         | DATE   | ○    | カレンダーピッカー or 手入力; デフォルト = クリック日                                                |
| シフトタイプ | RADIO  | ○    | NORMAL / LESSON / OTHER; デフォルト = NORMAL                                                         |
| 開始時刻     | TIME   | ◆    | NORMAL/OTHER型のみ表示; HH:MM 形式; バリデーション: 00:00 ～ 23:59                                   |
| 終了時刻     | TIME   | ◆    | NORMAL/OTHER型のみ表示; HH:MM 形式; 開始時刻と同時刻は不可。開始時刻より早い場合は翌日終了として扱う |
| 休憩時間     | NUMBER | ○    | 分単位; デフォルト = 0; 0 ～ 240                                                                     |
| コマ種別     | RADIO  | ◆    | LESSON型のみ表示; NORMAL（通常期）/ INTENSIVE（講習期）                                              |
| コマ範囲     | SELECT | ◆    | LESSON型のみ表示; Period 1～N; startPeriod ≤ endPeriod を確認                                        |
| コメント     | TEXT   | ー   | 任意; 最大100文字; 改行不可; 空文字・空白のみは NULL として保存                                      |

**説明**

- **◆ 条件付き必須**: シフトタイプに応じて表示・必須が変わる
  - NORMAL/OTHER型: `開始時刻`、`終了時刻` は必須。`コマ種別`、`コマ範囲` は非表示
  - LESSON型: `コマ種別`、`コマ範囲` は必須。`開始時刻`、`終了時刻` は非表示（Timetableから自動計算）
- コメント入力欄の下に Google Calendar のイベント名登録イメージをリアルタイム表示する。
  - comment 入力あり: `イベント名プレビュー「勤務先名 (コメント)」`
  - comment 未入力: `イベント名プレビュー「勤務先名」`
  - 勤務先未選択の場合は、プレビューを非表示にするか `勤務先を選択するとイベント名を確認できます` と表示する。

**バリデーション**

1. **必須チェック**: 条件付き必須項目を確認
2. **時刻バリデーション**:
   - NORMAL/OTHER型: 開始時刻と終了時刻は同時刻不可
   - NORMAL/OTHER型で終了時刻 < 開始時刻の場合、翌日終了として扱い、保存前に確認ダイアログを表示する
   - （警告）日跨ぎ解釈後の時間帯が既存シフトと重複している場合の警告
3. **コマ範囲**: LESSON型選択時、有効なコマが存在するか確認
4. **コメント**: 最大100文字、改行不可。空文字・空白のみは NULL

**日跨ぎ確認ダイアログ**

- 対象: NORMAL/OTHER型で終了時刻 < 開始時刻の場合
- 表示タイミング: 登録送信前
- 表示内容: 開始日時、終了日時、翌日終了として登録する旨
- アクション: `キャンセル` / `翌日終了として保存`

**SCR_005: シフト編集時の差分仕様**

- シフト編集フォームは SCR_004 と同じ時刻解釈を使用する。
- 編集後に終了時刻 < 開始時刻となる場合、保存前に日跨ぎ確認ダイアログを表示する。
- 既存シフトがすでに日跨ぎで、開始時刻・終了時刻を変更していない場合は確認を省略する。

| ERR_002      | 開始時刻と終了時刻は同じ時刻にできません   |
| ERR_003      | この日付にはすでにシフトが登録されています |
| ERR_004      | 塾の授業は時間割が登録されていません       |

| ERR_VAL_001  | 必須項目が空   | 「〇〇は必須項目です」                       | フォーカス移動 + 赤背景    |
| ERR_VAL_002  | メール形式不正 | 「正しいメールアドレスを入力してください」   | 入力値削除，フォーカス移動 |
| ERR_VAL_003  | 時刻が不正     | 「開始時刻と終了時刻は同じ時刻にできません」 | 入力値ハイライト           |
| ERR_VAL_004  | 数値範囲外     | 「〇〇は〇〇以上〇〇以下で入力してください」 | 入力値削除                 |

## 10.2 システムエラー（サーバー側）

| エラーコード | 原因                       | ユーザー表示                                   | 対応                        |
| ------------ | -------------------------- | ---------------------------------------------- | --------------------------- |
| SYS_001      | DB接続失敗                 | 「通信に失敗しました。もう一度お試しください」 | リトライボタン表示          |
| SYS_002      | Google Calendar API エラー | 「Googleカレンダーとの同期に失敗しました」     | ログ記録 + 部分成功状態記録 |
| SYS_003      | データ競合                 | 「別ユーザーが同時に更新しました」             | 画面リロード促し            |

## 10.3 エラーメッセージの表示

- **Must-fix エラー**: 赤トースト or インラインエラー表示
- **警告**: 黄トースト表示（上書き許容）
- **成功**: 緑トースト表示 → 2秒後に自動非表示

# 11. 同期・セキュリティ・レスポンシブ

## 11.1 Google Calendar 同期フロー

| シフト操作 | 同期アクション | イベント属性                                                           | 注記                 |
| ---------- | -------------- | ---------------------------------------------------------------------- | -------------------- |
| **作成**   | イベント作成   | title: comment ありは `勤務先名 (コメント)`、comment なしは `勤務先名` | googleEventId を保存 |
| **編集**   | イベント更新   | 時間・タイトルを更新。comment 削除時は `勤務先名` のみに戻す           | googleEventId で識別 |
| **削除**   | イベント削除   | googleEventId にて削除                                                 | DB削除と同期         |

**同期失敗時の処理**

- DB には既に保存済み
- ユーザーに「Googleカレンダーとの同期に失敗しました」と表示
- 詳細ログはサーバー側に記録
- 将来「同期再試行」機能で手動対応

## 11.2 権限・セキュリティ

### 認証

- NextAuth.js でメール認証
- セッションベース管理
- CSRF トークン自動付加

### 認可

- ログイン前: SCR_001 のみ表示
- ログイン後: 全画面にアクセス可能（本システムは単一ユーザー）
- API: `Authorization: Bearer {token}` または セッション 検証

## 11.3 レスポンシブデザイン

### PC（1024px 以上）

- 左パネルナビゲーション固定
- メインエリアは可変幅
- テーブルは横スクロール可

### タブレット（768px ～ 1023px）

- 左パネルはハンバーガーメニューに
- モーダルは画面幅 80%
- テーブルはカード表示に変更

### モバイル（767px 以下）

- フルスクリーン表示
- 左パネルはドロワーメニュー
- モーダルは全画面
- ボタンサイズ: 最小 44px
